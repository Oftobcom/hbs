Вы уже создали серьёзную гемодинамическую модель организма (система ОДУ с 22 переменными, включая 4-камерное сердце с переменной эластичностью). Модель Алиева-Панфилова (АП) решает совсем другую задачу – **распространение волны возбуждения в 2D/3D миокарде**. Однако её можно интегрировать в вашу систему несколькими способами – от простого обогащения существующей модели до полноценного гибридного моделирования.

Ниже – практические варианты, от самых простых к более сложным.

## Вариант 1. Замена эластансной модели на «умный» генератор частоты и инотропизма

**Идея:** Модель АП не будет решаться в пространстве, а только **в одной точке** (single‑cell режим). Вы получаете:
- длительность потенциала действия (APD) и рефрактерность как функцию предыдущего интервала (реституция);
- возможность имитировать экстрасистолы, чередование (alternans) и блокаду проведения.

**Как встроить в `Heart4Chambers`**:

Добавьте в `heart.py` новый модуль, который на каждом шаге интегрирования вычисляет текущую ЧСС и инотропный фактор на основе реституции АП.

```python
# Добавить в __init__ Heart4Chambers
self.ap_model = AlievPanfilovSingleCell(mu1=0.2, mu2=0.3)  # реализуем ниже
self.last_ap_trigger = 0.0

# Внутри get_derivatives, перед расчётом эластансов:
current_time = t
# Проверяем, пора ли генерировать новый потенциал?
# Например, по достижении порога P_la > threshold или по истечении рефрактерности
if self.ap_model.is_excitable(current_time):
    # Стимуляция может быть синусовой (по расписанию) или от соседнего участка
    self.ap_model.fire(current_time)
    # Получаем APD и рефрактерность
    apd = self.ap_model.get_apd()
    # Меняем ЧСС и инотропизм в зависимости от APD (короткий APD → высокая ЧСС)
    hr_factor = self.ap_model.compute_hr_factor()
    inotropy_factor = self.ap_model.compute_inotropy_factor()
    # Передаём в сердечную механику
    self._current_hr = self.hr_base * hr_factor
    self._current_E_max = {ch: self.E_max_base[ch] * inotropy_factor for ch in ...}
```

**Реализация одиночной клетки АП** (файл `aliev_panfilov_cell.py`):

```python
import numpy as np
from scipy.integrate import solve_ivp

class AlievPanfilovSingleCell:
    def __init__(self, k=8, a=0.15, e0=0.002, mu1=0.2, mu2=0.3):
        self.k = k
        self.a = a
        self.e0 = e0
        self.mu1 = mu1
        self.mu2 = mu2
        self.state = None
        self.reset()

    def reset(self):
        self.state = np.array([0.0, 0.0])  # u, v
        self.last_fire_time = -np.inf
        self.apd = None

    def rhs(self, t, y):
        u, v = y
        epsilon = self.e0 + self.mu1 * v / (u + self.mu2)
        du = -self.k * u * (u - self.a) * (u - 1) - u * v
        dv = epsilon * (-v - self.k * u * (u - self.a - 1))
        return [du, dv]

    def fire(self, t_peak):
        """Вызывается в момент стимуляции – устанавливаем u=1 (пик потенциала)"""
        self.state[0] = 1.0
        self.state[1] = 0.0
        self.last_fire_time = t_peak

    def is_excitable(self, t, refr_fraction=0.8):
        """Возвращает True, если клетка может ответить на стимул"""
        if self.last_fire_time == -np.inf:
            return True
        if self.apd is None:
            return True
        # Рефрактерность = APD * refr_fraction
        return (t - self.last_fire_time) > (self.apd * refr_fraction)

    def integrate(self, t_span, dt=0.5):
        """Проинтегрировать от last_fire_time до t_span[1]"""
        sol = solve_ivp(self.rhs, t_span, self.state, t_eval=np.arange(t_span[0], t_span[1], dt))
        self.state = sol.y[:, -1]
        # Определить APD по пересечению 90% реполяризации
        u = sol.y[0]
        t_eval = sol.t
        # Ищем момент, когда u становится <= 0.1 (от пика 1.0)
        idx = np.where(u <= 0.1)[0]
        if len(idx) > 0 and len(u) > 0 and u[0] > 0.9:
            repol_time = t_eval[idx[0]]
            self.apd = repol_time - self.last_fire_time
        return sol

    def compute_hr_factor(self):
        """Пример: обратная зависимость от APD (короткий APD → высокая ЧСС)"""
        if self.apd is None:
            return 1.0
        apd_norm = self.apd / 330.0  # 330 ms – APD0
        # При APD/APD0 = 0.53 (как в вихре) частота растёт в ~1.5 раза
        hr_factor = max(0.6, min(2.0, 1.0 / apd_norm))
        return hr_factor
```

**Преимущества:** минимальные изменения, добавляется физиологическая реституция.  
**Ограничение:** нет пространственных волн, нет аритмий, связанных с геометрией.

## Вариант 2. Гибридная модель: АП для электрики + эластансная модель для механики

Это популярный подход в вычислительной кардиологии. Вы решаете **2D/3D уравнение АП на конечно-элементной сетке миокарда** (например, на сетке желудочков), а затем пересчитываете активное напряжение (или изменение жёсткости) в каждый момент времени, чтобы управлять эластансной моделью сердца.

**Как связать с вашим `WholeBodyModel`**:

1. В отдельном потоке или на каждом шаге `solve_ivp` решаете PDE АП. Получаете распределение `u(x,y,z,t)`.
2. Вычисляете **среднюю активность в ЛЖ и ПЖ** – например, процент клеток в состоянии деполяризации.
3. Эта активность модулирует **эластичность** (E_max) в вашей lumped-модели:

```python
# В heart.py
def _update_parameters(self, t, inputs):
    # ... базовый расчёт hr_factor, inotropy_factor
    # Добавляем влияние электрической активности от модели АП
    if 'ap_activation_lv' in inputs:
        act_lv = inputs['ap_activation_lv']  # от 0 до 1
        # Увеличиваем E_max пропорционально активации (систола)
        self._current_E_max['LV'] = self.E_max_base['LV'] * (0.5 + act_lv)
    # Аналогично для ПЖ
```

4. Изменение давления в камерах (рассчитанное по эластансам) будет влиять на потоки и объёмы, которые через венозный возврат и артериальное давление влияют на периферию. Это даёт **электромеханическую обратную связь**.

**Сложность:** нужно реализовать численное решение PDE (метод конечных разностей или конечных элементов) на 2D/3D сетке. Можно использовать готовые библиотеки: `FiPy`, `FEniCS`, или написать свой простой явный схемы на `numpy`.

**Пример простого 2D решателя** (для демонстрации связи):

```python
# aliev_panfilov_2d.py
import numpy as np

def solve_ap_2d(ap_model, u, v, dx, dt, n_steps):
    """Простая явная схема реакция-диффузия на сетке NxN"""
    N = u.shape[0]
    D = 1.0  # коэффициент диффузии
    for _ in range(n_steps):
        # Лапласиан (5-точечная схема)
        laplacian = (np.roll(u, 1, axis=0) + np.roll(u, -1, axis=0) +
                     np.roll(u, 1, axis=1) + np.roll(u, -1, axis=1) - 4*u) / dx**2
        du = D * laplacian + ap_model.reaction(u, v)
        dv = ap_model.reaction_slow(u, v)
        u += dt * du
        v += dt * dv
    return u, v
```

## Вариант 3. Использование предвычисленных таблиц реституции

Если вам не нужны пространственные волны, а только правильная зависимость APD от предыдущего цикла – вы можете **предварительно вычислить реституционную кривую** с помощью одиночной клетки АП (Вариант 1), а затем использовать её как таблицу поиска в вашей модели сердца.

Этот подход:
- не добавляет дополнительных ОДУ в систему;
- даёт физиологически верное изменение длительности ПД и рефрактерности при изменениях ЧСС;
- может имитировать чередование (alternans) при быстрой стимуляции.

**Интеграция в `Heart4Chambers`**:
- Вместо `hr_factor` как функции времени вы храните предыдущие интервалы и по таблице определяете текущую APD, а затем через APD – возбудимость и сократимость.

## Вариант 4. Полная замена эластансной модели на электромеханическую модель сердца

Это самый амбициозный путь. Ваш `Heart4Chambers` полностью заменяется на 3D-модель, где:
- Электрическая волна (АП) распространяется по анатомической модели желудочков.
- Вычисляются активные напряжения (например, по модели Нидергера – Хантера), которые создают давление в камерах.
- Давления передаются в вашу системную гемодинамику (Windkessel).
- Объёмы камер, полученные из гемодинамики, задают граничные условия для механики (регулируют длину саркомера и предварительное напряжение).

**Такая интеграция** потребует:
- Использования специализированных библиотек (например, `CARPentry`, `OpenCMISS`, `LifeV`).
- Значительных вычислительных ресурсов (суперкомпьютер для 3D на несколько тысяч ядер).
- Но вы получите **полную физиологическую модель сердца**, способную генерировать фибрилляцию, реентри и электромеханическую диссинхронию.

## Что выбрать вам?

Учитывая вашу текущую реализацию (модель всего организма в `WholeBodyModel` как система ОДУ), я рекомендую начать с **Варианта 1 (одиночная клетка АП + модуляция ЧСС/инотропизма)**. Это даст немедленное улучшение физиологической точности вашей модели ДМЖП – например, при большом шунте тахикардия будет естественным образом влиять на реституцию и сократимость.

Если вы хотите изучать **пространственные аритмии** (фибрилляцию предсердий/желудочков), тогда переходите к **Варианту 2 (2D/3D электрика + обратная связь с гемодинамикой)**. Вы можете добавить в свой код небольшой 2D-солвер для одной поперечной среза желудочков.

Для быстрого прототипирования создайте отдельный класс `AlievPanfilovCoupling`, который вызывается из `derivatives` и возвращает `hr_factor`, `inotropy_factor`, а также флаг `is_fibrillation`. Это позволит вам имитировать переход от синусового ритма к аритмии при изменении `vsd_resistance`.

Добавьте в `WholeBodyModel.__init__`:

```python
self.ap_cell = AlievPanfilovSingleCell()

```

В `derivatives` перед расчётом сердца:

```python
# Обновляем состояние клетки АП
t_span_ap = (self.last_ecg_time, t)
if t_span_ap[1] > t_span_ap[0] + 1e-6:
    self.ap_cell.integrate(t_span_ap, dt=0.5)
    self.last_ecg_time = t

# Генерация естественного водителя ритма (например, каждые 800 мс)
if self.ap_cell.is_excitable(t) and (t - self.last_beat) > 0.8:
    self.ap_cell.fire(t)
    self.last_beat = t

hr_factor = self.ap_cell.compute_hr_factor()
heart_inputs['hr_factor'] = hr_factor
heart_inputs['inotropy_factor'] = max(0.5, min(1.5, 1.0 / hr_factor))
```

Теперь ваша гемодинамическая модель будет иметь частоту, зависящую от реституции, – намного более реалистично.
